## 1. PBO 在量什么:一个考试的比喻

把一次参数搜索想象成一场有 500 个考生的选拔:每个考生是一组参数,我们用
"模拟考成绩"(IS,优化器看过的历史数据)选出状元。

PBO(Probability of Backtest Overfitting,回测过拟合概率)问的是一个很扎心
的问题:**把试卷重新分成"模拟考/正式考"一万多种不同的分法,模拟考状元在
正式考上掉到中游以下的概率有多大?**

- 状元真有本事 → 怎么分卷子他都名列前茅 → **PBO ≈ 0**;
- 500 个考生水平其实差不多、状元只是运气好 → 换套卷子名次就是随机的 →
  **PBO ≈ 50%(抛硬币)**。

> **经验法则:PBO > 50% ⇒ 这次"选拔"大概率在挑噪声,选出来的"最优"不可信。**

机制(CSCV,组合对称交叉验证,Bailey-Prado 2017):把时间切成 S=16 块,
穷举 C(16,8)=12870 种"8 块当模拟考、8 块当正式考"的分法,每种分法都重演
一次"按 IS 选状元 → 看他 OOS 名次"。输出的不是单点,是 12870 条路径的分布。

**和 DSR 的分工**:DSR 折减的是**单个分数**里的多重检验运气("你考了 90 分,
但你考了 2500 次,90 分还稀奇吗");PBO 审计的是**"选拔"这个动作本身**
("就算分数都真实,按分数挑第一名这个做法靠不靠谱")。两把尺子互补。

## 2. 先校验尺子,再量东西

`cscv.py` 自带确定性自检:

| 喂给它什么 | 它报什么 | 说明 |
|---|---|---|
| 纯随机噪声的 200 个候选 | PBO = 0.58 | ≈ 50% 基线,符合"全是运气" |
| 同样的噪声 + 偷偷塞 1 个真学霸 | PBO = 0.000 | 有真东西时会亮绿灯 |

所以当它对真实数据报"≈噪声"时,这个读数是可信的——不是工具天生悲观。

## 3. 第一次实战:MRA 实验的 15 个选拔事件

对 MRA A/B 实验里每个 arm×fold 的 Optuna study(各 500 个 trial 在该折
IS 段的真实排名数据)做审计(数字权威:mra_ab README §13):

| arm | median PBO | degradation slope |
|---|---|---|
| A(现状镜像) | 40% | −0.94 … −0.98 |
| B(filter bank) | 39% | −0.51 … −0.79 |
| C(嵌套对照) | 47% | −0.47 … −0.88 |

### 读法一:三臂全部 ≈ 抛硬币

39%~47%,贴着噪声基线。意思是:优化器辛辛苦苦从 500 组参数里挑出的
"最优参数",和闭着眼睛随便抓一组,在新数据上没什么区别。**而且不只是
新模型 B——A 臂(照现役策略形态做的镜像)也一样。** 在小时线这个频率上,
"精调参数"这件事本身就没有多少可迁移的含金量:参数之间的成绩差异,
大部分是噪声。这与 MRA §12 的发现(净摩擦下三臂 OOS 全负)互相咬合:
信号弱 ⇒ 参数间真实差异小 ⇒ 选拔只剩噪声可挑。

### 读法二:slope 全负,比 PBO 更难看

把 500 个考生的"模拟考成绩 vs 正式考成绩"画散点图:本事是真的,线该往上
走(slope ≈ 1:模拟考好 → 正式考也好)。量出来**全是负的:模拟考考得越好,
正式考反而越差。**

为什么?在一堆水平相近的考生里,想在模拟考拿到"显眼的好成绩",唯一的
办法是把那套卷子的偶然规律背得特别死——背得越死,换卷子摔得越狠。
所以 **IS 排行榜的头部不是精英区,是噪声重灾区;挑第 1 名比挑第 250 名
还危险。**

### 一个容易误读的旁注

p(IS 冠军 OOS Sharpe < 0) 只有 ~3%——"挑噪声"不等于"会亏钱"。样本期偏牛,
随便哪组参数都被 beta 抬着;输掉的是**名次优势**(精调的那部分),不是
绝对生存。PBO 审计的是选拔,不是盈亏。

## 4. 两条实际含义

1. **对 MRA 研究线**:这是第三个独立角度(继 DSR 条款拦伪影、谱形 w2/w5
   反号之后)指向同一结论——不立项,row 4 判决很稳。
2. **更通用的警示**:在信号弱、摩擦贵的小时级场景,"回测排行榜第一名"是个
   危险头衔;一组中庸、稳定的参数比"冠军参数"更值得信。
   **边界**:这个警告目前只在简化模拟器 + 1h 粒度上量过;现役 champion 走的
   是四窗 crucible + OOS + DSR 整套闸门,不直接受此否定。"该不该用同样的
   尺子量一遍历史上每次 Promote"正是下半场(§5)。

## 5. 下半场:历史 challenger 审计(待路线裁决)

§12.4 item 4 的原题是"PBO 会不会推翻过任何一次 Promote"。卡点:PBO 需要
**T×N 收益序列矩阵**,而 `evaluation_traces` 只有单点分数。两条路线
(引擎级 RunReview 回放导出 = 保真优先,倾向;vs Python 简化镜像 = 便宜
但有已知保真度缺口)详见 `research/cpcv_pbo/README.md`。

一句话总结:**这把尺子量的不是"模型好不好",而是"'挑选'这个动作本身
是在挑本事还是挑运气"。MRA 实验里,三个臂的答案都是:基本在挑运气。**
